## 0. 핵심 원칙
어떤 프로세스도 최적화하기 전에 반드시 물어야 할 것:
1. **이 프로세스가 애초에 필요한가?**
2. **가능하면 제거하라.**
3. **그 다음에야 — 최적화하라.**

이 문서에 나오는 모든 전술, Tool/Tech, 회의에 이 테스트를 적용.

> **매주 스스로에게 물어야 할 질문:** *우리 사용자의 미래 가치는 무엇이고, 어떻게 예측하고 선제적으로 대응할 수 있는가?*

- 운영 업무는 위임.
- 사용자 생애 가치(LTV) 예측에 집중.

## 6. 직원 공감대 형성 프레임워크

1. **투명성:** 마일스톤 맵을 전 직원과 공개적으로 공유한다.
2. **오너십:** 모든 KPI에는 이름이 붙은 책임자가 있다 — 공동 책임은 아무도 책임지지 않는 것과 같다.
3. **인정:** 마일스톤 달성을 Slack (#wins) 에서 공개적으로 축하한다.
4. **피드백 루프:** 월간 회고는 비난의 장이 아닌 심리적 안전의 공간으로 운영한다.
5. **인센티브:** 월 $100만 달성이 팀원에게 무엇을 의미하는지 명확히 정의한다 (보너스, 지분 등).
